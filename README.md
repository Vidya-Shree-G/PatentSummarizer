# Patent Summarizer

End-to-end pipeline that clusters, summarizes and visualizes large patent corpora
using **TF-IDF · LDA · Sentence-BERT · K-Means · Hierarchical Clustering · t-SNE**.

**Group 22 · Project 9 · CSE 573 Spring 2026 · Arizona State University**

---

## Repository Structure

```
PatentSummarizer/
├── CODE/                          <- All pipeline source code
│   ├── main.py                    <- Pipeline entry point
│   ├── ingestion.py               <- Data loading (synthetic + PatentsView + bigpatent)
│   ├── preprocessing.py           <- Text cleaning & tokenization
│   ├── features.py                <- TF-IDF/LSA, LDA, Sentence-BERT embeddings
│   ├── clustering.py              <- K-Means++, Hierarchical, metrics, optimal-k
│   ├── summarization.py           <- Extractive summaries, ROUGE, keyword coverage
│   ├── visualization.py           <- t-SNE, PCA, topic bars, heatmaps, charts
│   └── report.py                  <- Self-contained HTML report generator
│
├── DATA/                          <- Datasets
│   ├── sample_patents.csv         <- 48 synthetic patents across 6 tech domains
│   └── real_patents.csv           <- 5,000 real USPTO patents (HuggingFace big_patent)
│
├── EVALUATIONS/                   <- Pipeline outputs (generated by running main.py)
│   ├── patent_analysis_report.html  <- Full interactive HTML report
│   ├── patent_clusters.csv          <- Patent-to-cluster assignments
│   ├── results.json                 <- Metrics + summaries (used by Streamlit app)
│   └── *.png                        <- 15 visualization figures
│   └── 10000/                       <- Pre-generated run for 10,000 patents (see below)
│
├── app.py                         <- Streamlit demo web app
├── Dockerfile                     <- Containerized run environment
├── docker-compose.yml             <- Three pre-configured run modes
└── requirements.txt               <- Python dependencies
```

## Quickstart

### Docker

```bash
# 1. Clone and enter the repo
git clone <repo-url>
cd PatentSummarizer

# 2. Build the image (first time, ~5 min)
docker compose build

# 3. Run the pipeline
docker compose up patent-pipeline
```

Results appear in `./EVALUATIONS/`.

### Local Python

```bash
# Run from the repo root (not from inside CODE/)
py -3.11 CODE/main.py --source synthetic --n 48 --k 6 --topics 6
```

### Streamlit Web Demo

Interactive web UI for live demos:

```bash
py -3.11 -m streamlit run app.py
```

Opens at `http://localhost:8501`. Features:

- **View saved results** — instantly view all pre-computed metrics, charts, and cluster summaries
- **Run pipeline live** — pick source / k / topics in the sidebar and watch the pipeline run in real time

## All Run Options

### Docker Compose

```bash
# Synthetic corpus (offline, 48 patents — default, ~10s)
docker compose up patent-pipeline

# Real USPTO patents from HuggingFace big_patent (5,000 patents, ~6 min)
docker compose up patent-pipeline-real

# Quick smoke-test (no charts, no report)
docker compose up patent-pipeline-test
```

### Local Python

```bash
# Synthetic corpus (fast, offline)
py -3.11 CODE/main.py --source synthetic --n 48 --k 6 --topics 6

# Real USPTO patents via HuggingFace big_patent, 5,000 docs, 5 clusters
py -3.11 CODE/main.py --source bigpatent --n 5000 --k 5 --topics 5

# Smaller scale (faster demo, ~30s)
py -3.11 CODE/main.py --source bigpatent --n 500 --k 5 --topics 5

# Large scale (10,000 patents; long run; consider --no-viz to speed up)
py -3.11 CODE/main.py --source bigpatent --n 10000 --k 6 --topics 6 --no-viz

# Skip visualizations (faster)
py -3.11 CODE/main.py --no-viz --no-report
```

## CLI Arguments

| Argument      | Default     | Description                                                          |
| ------------- | ----------- | -------------------------------------------------------------------- |
| `--source`    | `synthetic` | `synthetic` (48 offline) / `bigpatent` (HuggingFace) / `patentsview` |
| `--n`         | `48`        | Number of patents to load                                            |
| `--k`         | `6`         | Number of clusters (overrides silhouette-optimal k)                  |
| `--topics`    | `6`         | Number of LDA topics                                                 |
| `--no-viz`    | off         | Skip all chart generation                                            |
| `--no-report` | off         | Skip HTML report                                                     |

| Domain                          | CPC Code | Count |

are cached under `EVALUATIONS/.cache/` so repeat runs skip encoding.

**Full-scale evaluation (10,000 patents)** — used for current results and the

## Pipeline Architecture

```
Raw Patents (USPTO via big_patent / PatentsView / synthetic)
    |
    v
[CODE/ingestion.py]      Load & structure patent records
    |
    v
[CODE/preprocessing.py]  Clean -> Tokenize -> Remove stopwords
    |
    +-----------------------------+
    v                             v
[TF-IDF + LSA]           [Sentence-BERT]
[LDA Topic Model]         dense 384-dim embeddings
    |                             |
    +------------+----------------+
                 v
    [CODE/clustering.py]
    K-Means++ | Hierarchical (Ward)
    Silhouette | Davies-Bouldin | Stability ARI
                 |
       +---------+---------+
       v         v         v
  [Metrics]  [CODE/summarization.py]  [CODE/visualization.py]
  Silhouette  Keywords   t-SNE / PCA
  DB Index    ROUGE-1/2/L  Topic bars
  Stability   Coverage     Heatmap
              Centroid Sim
                 |
                 v
    [CODE/report.py] -> EVALUATIONS/patent_analysis_report.html
```

## Troubleshooting

**PatentsView returns no results**
The API has been intermittently broken. Use `--source bigpatent`
(real USPTO patents via HuggingFace) or `--source synthetic` instead.

**Out-of-memory at large scale**
The pipeline auto-scales: above 2,000 docs it switches TF-IDF to unigrams,
above 6,000 it skips Hierarchical Ward clustering. SBERT embeddings are
cached to `EVALUATIONS/.cache/` to skip the encoding step on repeat runs.
Close other apps if you have <8GB RAM and need to run at full 5,000 scale.

**Permission denied on EVALUATIONS/ folder (Linux)**

```bash
mkdir -p EVALUATIONS
chmod 777 EVALUATIONS
docker compose up patent-pipeline
```

**Docker Desktop memory (Windows)**
In Docker Desktop -> Settings -> Resources, allocate at least **4 GB RAM**
(Sentence-BERT requires ~400 MB).
